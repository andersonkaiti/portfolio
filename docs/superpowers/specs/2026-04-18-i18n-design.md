# i18n Design — en-US & pt-BR

**Date:** 2026-04-18  
**Status:** Approved

---

## Overview

Add full internationalization to the portfolio with support for English (en-US) and Brazilian Portuguese (pt-BR). The locale is stored in a cookie (`NEXT_LOCALE`), detected from the browser on first visit, and toggled via a button in the navbar. No URL prefixes — the app URL stays the same regardless of language.

---

## Architecture

**Library:** next-intl in "no routing" mode (cookie-based locale, no URL segments).

**Locale resolution order:**

1. Cookie `NEXT_LOCALE` (set by user toggle)
2. `Accept-Language` request header (browser default on first visit)
3. Fallback: `en`

**Supported locales:** `en`, `pt-BR`  
**Default locale:** detected from browser, fallback `en`

**Key files:**

- `messages/en.json` — all English strings
- `messages/pt-BR.json` — all Portuguese strings
- `src/i18n/routing.ts` — defines supported locales and default
- `src/i18n/request.ts` — reads cookie + Accept-Language, returns locale to next-intl
- `src/middleware.ts` — next-intl middleware for cookie-based locale

---

## Translation Scope

All visible text content is translated, including:

- **Navigation:** nav item labels (Introduction, About, Experiences, Projects, Technologies, Education, Contact)
- **Header section:** availability badge, title, subtitle paragraph
- **About section:** section labels/titles, both paragraphs
- **Experiences section:** section labels/titles, all position titles, employment types, periods, and full description bullets
- **Technologies section:** section labels/titles
- **Projects section:** section labels/titles, project UI strings (search, filter, etc.)
- **Education section:** section labels/titles, card content (degree names, descriptions, badge labels)
- **Footer/Contact section:** section labels/titles, button text, tagline
- **Page metadata:** `title`, `description`, `openGraph`, `twitter` — generated via `generateMetadata()` per locale

The `experiences.ts` data file becomes a function `getWorkExperiences(t)` that receives the `t` translator and returns the typed array with translated strings.

## Language Toggle

`LanguageToggle` is a `'use client'` component placed in the navbar next to `DarkModeButton`. It displays the current locale label (`EN` / `PT`). On click, it writes `NEXT_LOCALE=<newLocale>` to `document.cookie` with `path=/; max-age=31536000` and calls `router.refresh()` to re-render Server Components with the new locale — no full page reload.

Visual style: consistent with `DarkModeButton` (ghost button variant, border).

---

## Data Flow

```txt
HTTP Request
  → src/middleware.ts (next-intl) reads NEXT_LOCALE cookie or Accept-Language
  → locale written to next-intl internal context
  → src/i18n/request.ts resolves messages file
  → Server Components call getTranslations(namespace)
  → NextIntlClientProvider passes locale + messages to client tree
  → Client Components call useTranslations(namespace)
  → LanguageToggle writes cookie + router.refresh() on locale change
  → Server re-renders with new locale, no URL change
```

## Error Handling

- Missing translation key: next-intl logs a warning in dev and falls back to the key name — no crash.
- Invalid locale in cookie: `src/i18n/request.ts` validates against the supported list before using; invalid values fall back to `en`.
- `Accept-Language` absent: falls back to `en`.

---

## Out of Scope

- More than 2 locales
- URL-based routing (`/en/`, `/pt-br/`)
- Pluralization or date/number formatting (not needed for portfolio content)
- Per-project description translation (GitHub API content, not static)
